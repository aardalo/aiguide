# Story: STORY-013 Let user prioritize and select stay options

## Metadata
- **Priority**: P1
- **Status**: planned

## User story
As a planner, I want to reorder and prioritize suggested stay options, so that my personal preferences determine final selection order.

## Mode ownership
- Primary mode: PLAN
- Secondary modes: RUN

## Acceptance criteria
- [ ] User can reorder option list.
- [ ] Priority order persists per trip day.
- [ ] User can mark one option as selected candidate.

## Dependencies
- Option list UI and persistence model
